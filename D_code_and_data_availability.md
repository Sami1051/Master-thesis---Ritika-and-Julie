## D.3 Data files — verified independently

Both processed data files have been checked directly, not just assumed correct:

- **`ETS_price_clean_full.csv`** (1,700 rows, 2018-01-08 to 2025-12-15, columns `Date`, `price_eur`).
  Filtering to Phase 4 (≥ 2021-01-01) as the notebook does gives exactly 1,079 observations spanning
  2021-01-29 to 2025-12-15 — matching the notebook's own reported sample. Re-running the GBM
  calibration (`calibrate_gbm`) directly against this file reproduces μ=0.3062/yr, σ=0.4226/yr,
  matching the notebook's printed output (μ=0.306, σ=0.423) to four decimal places. This is also
  demonstrated as an executable check inside Appendix B, §B.7.
- **`NO1_hourly_prices_raw.csv`** (32,664 hourly rows, 2022-10-01 to 2026-06-22, columns `time_start`,
  `NOK_per_kWh`, `EUR_per_kWh`). Aggregating to daily averages and re-running the OU calibration
  (`calibrate_ou_electricity`) reproduces θ=31.688/yr, μ=0.785 NOK/kWh, σ=4.283, matching the
  notebook's printed output exactly.

Both stochastic-process calibrations behind the thesis's Monte Carlo results are therefore
independently reproducible from the raw data included in this repository — not dependent on trusting
a console printout from a single run.

## D.5 Environment

`requirements.txt` pins the package versions this repository's notebooks were verified to run under
(`numpy`, `pandas`, `scipy`, `matplotlib`, plus `jupyter`/`nbformat` for execution). If the thesis's
original results were generated in a different environment, replace these with the output of
`pip freeze` run from that original environment, so the pinned versions match what actually produced
the reported numbers rather than only versions known to work.
